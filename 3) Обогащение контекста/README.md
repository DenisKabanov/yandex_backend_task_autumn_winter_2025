# Обогащение контекста


## Задание:
Пользователи последних больших языковых моделей начали сталкиваться с проблемой - модель, сохраняя контекст в виде слов, может потреблять слишком много ресурсов. Конечно, эту проблему можно решить, найдя лучшее оборудование, но исследователи задались вопросом: а как сжать контекст, не потеряв точности? Ответ, к которому они пришли - **объединять в один контекст вопросы пользователя, пересекающиеся хотя бы по одному слову**. В качестве наглядных метрик они выбрали количество таких различных контекстов после всех запросов и размер (в уникальных словах) наибольшего из них. Например, если запросы пользователя - это "first prompt" и "second prompt", то они объединяются в один контекст размера 3, состоящий из слов "first", "second" и "prompt". Помогите исследователям реализовать эффективный алгоритм сжатия контекста.


## Формат ввода (стандартный ввод или input.txt):
В первой строке вводится число ***N*** (*1 ≤ N ≤ 1000*) - количество запросов от пользователя.<br>
Каждый запрос состоит из двух строк: 
* на первой дано число ***$M_i$*** (*1 ≤ *$M_i$* ≤ 1000*) - количество слов в *i*-м запросе;
* вторая строка - это сам запрос, включающий $M_i$ слов из строчных английских букв длины от *1* до *10*, разделенных пробелом.


## Формат вывода (стандартный вывод или output.txt):
В одной строке выведите два числа через пробел - кол-во контекстов и размер (в словах) самого большого из них.


## Примеры:
1) 
    **Ввод:**
    ```
    3
    4
    a four word prompt
    2
    small prompt
    5
    unique line with five words
    ```

    **Вывод:**
    ```
    2 5
    ```
    *Примечание.* Первый запрос объединится со вторым, так как у них есть общее слово "prompt", из-за чего их длина станер равной 5 ("a", "four", "word", "prompt", "small"), что совпадает с длиной отдельного третьего запроса.
---

2) 
    **Ввод:**
    ```
    1
    1
    a
    ```

    **Вывод:**
    ```
    1 1
    ```
    *Примечание.* Имеется всего один запрос из одного слова "a".
---

3) 
    **Ввод:**
    ```
    2
    2
    first prompt
    2
    second prompt
    ```

    **Вывод:**
    ```
    1 3
    ```
    *Примечание.* При объединении останется один запрос длины 3 ("first", "prompt", "second").
